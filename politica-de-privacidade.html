Política de Privacidade

1. Introdução

A equipe do Katalog valoriza a privacidade e a segurança dos dados pessoais de seus usuários. Esta Política de Privacidade tem como objetivo informar de maneira clara e transparente como os dados são coletados, utilizados, armazenados e protegidos dentro da plataforma.

O Katalog é um sistema web desenvolvido para organizar, catalogar e compartilhar coleções musicais, oferecendo aos colecionadores um ambiente seguro, colaborativo e acessível. O tratamento das informações segue rigorosamente a Lei nº 13.709/2018 – Lei Geral de Proteção de Dados (LGPD).

2. Dados Coletados

O sistema coleta apenas os dados estritamente necessários para o funcionamento da plataforma, que incluem:

  • Nome e endereço de e-mail (para autenticação e identificação da conta via Firebase Authentication);
  • Preferências de interface (tema claro/escuro, armazenado localmente no navegador via localStorage);
  • Informações das coleções cadastradas (álbuns, artistas, formatos, capas, ano, número de catálogo e descrições);
  • Status de verificação de e-mail (para controle de limites de importação);
  • Contadores de importação (para usuários não verificados, armazenado localmente via localStorage);
  • Dados técnicos de autenticação (tokens de sessão gerenciados automaticamente pelo Firebase Authentication).

Importante: O Katalog NÃO coleta dados sensíveis como informações financeiras, biométricas, localização precisa ou histórico de navegação.

3. Finalidade do Tratamento de Dados

Os dados coletados são utilizados exclusivamente para as seguintes finalidades:

  • Gerenciar a conta do usuário e permitir seu acesso seguro ao sistema;
  • Armazenar e organizar coleções musicais de forma individualizada;
  • Permitir a personalização da experiência (tema claro/escuro);
  • Controlar limites de importação para usuários não verificados;
  • Garantir a segurança da conta através de verificação de e-mail;
  • Viabilizar o compartilhamento público de coleções, quando autorizado pelo usuário;
  • Buscar automaticamente capas de álbuns através de APIs externas (iTunes, quando solicitado pelo usuário).

O tratamento de dados é sempre realizado com base na finalidade legítima e no consentimento do usuário, em conformidade com o artigo 7º da LGPD.

4. Armazenamento e Segurança das Informações

4.1 Infraestrutura em Nuvem

O Katalog utiliza a infraestrutura segura do Google Firebase (Google Cloud Platform) para o armazenamento e processamento das informações. Essa tecnologia garante:

  • Criptografia de dados em trânsito (HTTPS/TLS) e em repouso;
  • Autenticação por token em cada sessão de usuário;
  • Controle de acesso individual baseado no identificador único (UID);
  • Regras de segurança no Firestore que restringem o acesso aos dados apenas ao seu proprietário;
  • Backups automáticos para garantir disponibilidade e recuperação das informações.

4.2 Armazenamento Local (localStorage)

Algumas preferências são armazenadas localmente no navegador do usuário através da tecnologia localStorage:

  • Tema da interface (claro/escuro) - chave: katalog-theme
  • Contadores de importação (somente para usuários não verificados) - chave: import_tracking_[userId]

Observação: Dados do localStorage ficam apenas no seu dispositivo e não são enviados para servidores. Limpar o cache do navegador remove essas preferências.

5. Uso de APIs e Serviços Externos

Para melhorar a experiência do usuário, o Katalog utiliza os seguintes serviços externos:

iTunes Search API (Apple Inc.)

Finalidade: Buscar automaticamente capas de álbuns, correção de nomes de artistas e anos de lançamento.
Dados enviados: Nome do álbum e artista (enviados temporariamente para consulta).
Política de Privacidade: https://www.apple.com/legal/privacy/

Google Images

Finalidade: Permitir busca manual de capas de álbuns (abre em nova aba).
Dados enviados: Nenhum dado pessoal. Apenas termos de busca (nome do álbum).
Observação: O usuário é redirecionado para o Google, sujeito à política de privacidade do Google.

CDNs Públicos

Serviços: Tailwind CSS, Lucide Icons, Alpine.js, SheetJS, QRCode.js
Finalidade: Carregamento de bibliotecas JavaScript e estilos.
Dados: Apenas requisições HTTP padrão (sem dados pessoais).

Importante: O Katalog NÃO tem controle sobre a política de privacidade de serviços externos. Recomendamos a leitura das políticas de cada serviço.

6. Compartilhamento de Dados

O Katalog NÃO compartilha, vende ou aluga informações pessoais a terceiros para fins comerciais ou publicitários.

O único tipo de compartilhamento possível ocorre quando o próprio usuário opta por tornar sua coleção pública, gerando um link (formato: ?shared=[userId]) ou QR Code que permite visualização em modo somente leitura.

Coleções Públicas: Quando ativadas, qualquer pessoa com o link pode visualizar os álbuns, artistas e capas da coleção. Dados de e-mail e preferências permanecem privados.

7. Uso de Cookies e Tecnologias Similares

O Katalog NÃO utiliza cookies tradicionais para armazenamento de preferências ou rastreamento de navegação.

Em vez disso, o sistema utiliza:

  • localStorage (armazenamento local do navegador) para salvar tema e contadores de importação;
  • Firebase Authentication Tokens (gerenciados automaticamente pelo Firebase) para manter a sessão do usuário autenticada de forma segura.

Atenção: O Katalog NÃO requer banner de consentimento de cookies, pois não utiliza cookies para rastreamento, publicidade ou análise comportamental.

8. Direitos do Usuário

De acordo com a LGPD, o usuário possui os seguintes direitos:

  • Confirmar a existência de tratamento de dados pessoais;
  • Acessar seus dados a qualquer momento através do painel de configurações;
  • Corrigir, atualizar ou completar informações (nome, senha, dados da coleção);
  • Solicitar a portabilidade dos dados (exportação para Excel);
  • Revogar o consentimento de compartilhamento público;
  • Solicitar a exclusão completa da conta e de todos os dados associados.

Esses direitos podem ser exercidos diretamente no painel de configurações do sistema, sem necessidade de contato intermediário ou justificativa.

9. Exclusão de Conta e Retenção de Dados

Quando o usuário solicita a exclusão da conta através do painel de configurações, o sistema realiza:

  1. Exclusão permanente de todos os registros da coleção (álbuns, artistas, capas);
  2. Remoção dos dados de perfil (nome, preferências);
  3. Exclusão da conta de autenticação no Firebase;
  4. Limpeza automática do localStorage no navegador.

Atenção: A exclusão é permanente e irreversível. Nenhuma cópia é mantida após o encerramento, garantindo o direito ao esquecimento previsto na LGPD.

10. Limitações para Usuários Não Verificados

Para prevenir abuso do sistema, usuários que não verificaram o endereço de e-mail possuem as seguintes limitações:

  • Máximo de 2 importações de arquivo Excel/CSV;
  • Máximo de 100 registros por importação;
  • Contadores armazenados localmente (localStorage) para controle.

Após a verificação do e-mail, todas as limitações são removidas automaticamente.

11. Alterações na Política de Privacidade

Esta Política de Privacidade poderá ser atualizada periodicamente, sempre que houver modificações significativas na forma de tratamento dos dados. As alterações serão informadas dentro da própria plataforma, e o uso contínuo do sistema após as atualizações implicará na aceitação dos novos termos.

A equipe do Katalog reafirma seu compromisso com a segurança, a ética e a privacidade de todos os usuários, adotando boas práticas de segurança digital e garantindo total transparência sobre o uso e a proteção das informações pessoais.
